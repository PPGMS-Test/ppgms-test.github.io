8. 核心设计哲学总结

SDK v6（jsv6-test）

  • Run-time 配置，不依赖 URL 参数
  • client-id 完全隐藏在后端
  • 开发者完全控制 UI，SDK 仅处理支付逻辑
  • presentationMode 显式控制弹窗 / 重定向
  • 多币种无需重新加载脚本
  • 适合：需要强安全性和 UI 控制权的场景

SDK v5（GMS-exhibition）

  • Load-time 配置，所有参数在 URL 中声明
  • client-id 前端可见
  • SDK 自动接管完整 UI 流程
  • 完整 Vault 生命周期（存储 + 重复使用）
  • 丰富的 payment_source 控制
  • 适合：快速集成、需要 Vault 功能的场景
