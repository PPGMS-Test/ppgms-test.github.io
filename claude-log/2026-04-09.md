# Claude Log

## 2026-04-09 — paylater-msg-dashboard.html

### 背景
`src/pages/[3]-paylater/PLM-admin-dashboard/paylater-msg-dashboard.html` 是一个用于测试 PayPal Pay Later Message 在各国资格的工具页面。

### 问题与决策

**问题1：某些国家偶发 Timeout**
- 现象：使用有效 client-id，8个国家理论上都有权限，但多次测试后某几个国家随机出现 `Timeout - SDK did not respond`
- 原因：PayPal Messages SDK 在并发渲染多个实例时，`onRender` 回调偶尔不触发
- 解决：加入重试机制。第一次超时只记 log（`retrying...`），自动重试一次，第二次失败才显示在卡片上

**问题2：国家无权限时无法捕获错误**
- 现象：无权限时 SDK 抛出 `paypal_messages_buyer_country_not_authorized`，错误对象结构为 `{description: 'For production use of buyer country...'}`
- 该错误**不在 `.render().catch()` 里**，而是作为 script 级错误抛出
- 尝试方案：MutationObserver 监听 iframe 插入 → **失败，回退**（SDK 渲染方式不确定，导致页面报错）
- 最终方案：`.catch()` 里取 `err?.description || err?.message`，捕获能拿到的部分；script 级错误目前无法在并发模式下准确归因到具体国家，依赖超时兜底+重试覆盖

**问题3：并发 vs 串行**
- 串行（`for...of`）可配合全局错误监听准确归因，但速度慢（最坏 8 国 × 8s）
- 最终保留并发（`Promise.all`），配合重试机制降低假超时概率

### 当前核心逻辑
```
checkCountryWithRetry(country)
  └── checkCountry()          第一次
        onRender → success
        .catch()  → failed（立即）
        setTimeout 8s → timeout（兜底）
  └── 若第一次 timeout → log + checkCountry() 第二次
        同上，结果直接显示到卡片
```
